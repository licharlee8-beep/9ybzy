# 久元网站 — 写作规范

## 写作风格

- **多用数据说话**：每个观点尽量搭配具体数据（百分比、金额、案例年份、人数等），避免空洞的形容词堆砌。例如不说"很多考生滑档"，而说"近3年12.7%的滑档案例与志愿泄露有关"。
- **表格对比**：方案对比、机构对比等用表格呈现，数据一目了然。
- **真实案例**：适当使用脱敏后的真实案例（如"2024年河南一位理科考生..."），增强可信度。

## 每日备份规则

每天第一次修改代码前，先执行以下备份操作：

1. 运行备份脚本：`bash .claude/backup.sh`
2. 脚本功能：将当前 `index.html` 复制到 `.claude/backup-index.html`
3. 每天只备份一次，第二天再次备份时覆盖前一天的备份
4. 如需回滚：`cp .claude/backup-index.html index.html`

## 文章发布规则

每次新增文章后，必须同步完成以下所有操作：

1. **创建 HTML 文件** — `articles/YYYY-MM-DD-slug.html`
2. **更新首页侧栏「最新文章」** — 只保留最近5篇，新文章放最上面，删掉最早的第6篇及之后
3. **更新 articles/index.html 全部文章列表** — 新文章放最上面
4. **更新 sitemap.xml** — 添加新文章的 URL 条目（放首页之后、其他文章之前）
5. **更新 llms.txt** — 在「文章」章节头部添加新文章链接和摘要
6. **提交并推送** — `git add -A && git commit && git push`
